# Разделение `/track` на `/state` и `/report`

**Дата:** 2026-07-01

## Проблема

Единая команда `/track` смешивает субъективное состояние (меняется в течение дня)
и дневные факты-итоги (раз в день). Хочется:

1. Трекать состояние **несколько раз в день** (утро/обед/вечер), без усреднений —
   каждая запись это отдельная строка в дневном файле с меткой времени.
2. Отделить дневной отчёт (сон, VPN, English, вес, лекарства) в свою команду.
3. Убрать тревогу и раздражительность полностью.
4. В отчёте **не спрашивать** про таблетки, время приёма которых ещё не наступило.

## Решение

### Две команды

- **`/state`** — настроение, энергия, аппетит. Многоразовая. Без resume — каждый
  запуск это новый снимок «сейчас». В конце пишет новую строку `StateEntry` с
  текущим локальным временем. Короткий флоу из 3 вопросов (кнопки + текст).
- **`/report`** — сон(ч), лекарства, VPN(ч), English(ч), вес (сб). Раз в день.
  Сохраняет write-through + resume (можно прервать и дозаполнить). Тревога/
  раздражительность выпилены.

### Фильтр таблеток по времени в `/report`

Спрашиваем только про meds, у которых `reminder_time is None` **или**
`reminder_time <= сейчас` (локальное `HH:MM`). Таблетка на 11:00 при запуске в
09:00 не спрашивается; при повторном запуске в 12:00 — уже спрашивается (resume
пересчитывает pending по наступившему времени).

### База

Новая таблица (много строк на день):

```
StateEntry(id pk autoincrement, day: date, ts: datetime, mood, energy, appetite)
```

`day` = `subjective_today()`, `ts` = локальное время запуска (метка `HH:MM`).

`MoodEntry` остаётся отчётной таблицей (одна строка/день); в работе только
`sleep_hours, weight, vpn_hours, eng_hours` + `MedicationLog`. Колонки
`mood/anxiety/irritability/energy/appetite` не удаляются (nullable, без
деструктивной миграции), просто перестают использоваться. Одна Alembic-миграция
на `state_entries`.

### Раскладка в markdown

- **`## Самочувствие`** → лог состояния, по строке на запуск:
  ```
  - 09:15 — настроение: +1, энергия: 0, аппетит: +1
  - 14:30 — настроение: +2, энергия: +1, аппетит: −1
  ```
- **`## Отчёт`** (новая секция) → `- Сон (ч):`, `- Вес:` (сб), лекарства
  (`- Циталопрам: ✓ 20` / `✗`).
- **`## Время (ч)`** → VPN/Английский, in-place, как сейчас.

### Флаш и пинги

- `/state` буферизуется в SQLite; `flush_day` в 03:00 материализует все
  `StateEntry` за вчера строками в `## Самочувствие` (двухуровневая архитектура
  сохранена — GitHub source of truth).
- `flush_day` читает `MoodEntry` + `MedicationLog` → `## Отчёт` + `## Время (ч)`.
  Флаш не должен пропускаться, если есть хоть `StateEntry` **или** `MoodEntry`.
- `evening_ping` (21:00) нудит про `/report`, если он ещё не сделан (прокси —
  `MoodEntry.sleep_hours is None`).
- `/sync` без изменений (зовёт `flush_day`).

## Затрагиваемые файлы

- `db/models.py` — `StateEntry`; `alembic/versions/*` — миграция.
- `markdown/daily.py` — рендер лога состояния + секции отчёта; шаблон дня
  (добавить `## Отчёт`).
- `bot/handlers/state.py` (новый) — `/state`.
- `bot/handlers/report.py` (переименование `track.py`) — `/report` без mood-шагов,
  meds с фильтром по времени.
- `jobs/flush_day.py` — чтение `StateEntry` + рендер обеих секций.
- `jobs/scheduler.py` — `evening_ping`.
- `bot/app.py` — роутеры + `BOT_COMMANDS`.
- `bot/handlers/start.py`, `bot/handlers/today.py` — тексты.
- `tests/` — переписать track-тесты на report, добавить state + рендер + flush.
